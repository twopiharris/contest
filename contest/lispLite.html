Problem 7: Lisp Lite

Background

For this contest, you are most likely programming in an imperative langauge. However, there are several other completely different styles of languages. One of the oldest and most important is called 'functional languages.' Of these, a very old language called 'LISP' is probably the most important. LISP is a list-processing language. In LISP, everything is a list. What would be considered a command in another language is simply a list in LISP. Often the first element in a list is some sort of instruction. For example, to add two elements in LISP, you might say something like this: (+ 1 2)

This will add one and two, resulting in 3. You can nest instructions inside each other, so you can have more complex instructions like this: 

(+ (* 2 5) (- 15 10))

Inner lists evaluate first, so this expression would evaluate as (+ 10 5), and then 15.

Your job is to build a simple lisp-style interpreter with the following limitations:

  • You will have only four arithmetic operations: "+", "-", "*", "/".
  • Each instruction list will contain three elements.
  • The first will be an operation stored as a single character string.
  • The second and third element will be an operand.
  • Operands will either be integers or a list containing a new instruction.
  • You will not know how deeply the instructions are nested.
  • You can assume none of the input will lead to division by zero.
  • You can also assume there are no syntax errors in the input lines.

Input Expectations

Each line of input will be one expression in pseudo-lisp. Note that every token (including parentheses) will be followed by a space for easier parsing. There may be any number of input lines.

Please use "7in.txt" as the input file name.

Sample Input

( + 8 ( * ( - 5 3 ) ( / 10 2 ) ) )
( + 5 ( * 3 5 ) )

Output Expectations

Each line of input will generate a corresponding line of output. The output for each expression will be a single number. (Don't worry about whether the output is an integer or a float, as long as it is accurate.)

Sample Output

18
20
