API Treasure Hunt

Overview

This year's programming contest will be a bit different. We are doing a hack-a-thon. These are fun events that encourage you to learn new programming skills over time in a team.

This year's theme is 'API Treasure Hunt.' APIs are Application Programming Interfaces. In particular, there has been explosive growth of online APIs which produce data to use in other programs. We have provided you a list of various APIs you can explore. Your challenge is to write a program that does something interesting with the data you find online. The options are nearly limitless, with everything from weather forecasting to trivia questions. You ought to be able to find something interesting to write a program about.

How API programming works

The APIs we will explore are all based on web requests. For any of the databases, you can enter a query into an ordinary browser, and you will get a result. for example, try clicking this link. You will get a randomly-generated joke about programming. When you look at the associated documentation, you'll learn that you can change the various elements in the address to change what category, how many jokes you get, and so on.

The result usually comes back in a format called 'JSON.' (JavaScript Object Notation). This format is human-readable, and can also be read by most programming languages.

Once you've chosen an API, you will write a program that can:

  • Ask the user for various parameters
  • Generate an appropriate URL string
  • Pass the string to the API
  • Read the result string
  • Process the data
  • Print some kind of output

Suggested APIS

Please begin by looking at the following APIs. They have been selected from a larger list of publicly available APIs, with the following criterion:

  • Interesting content
  • No authentication tokens or logins are required
  • High-school appropriate content
  • All data is free and open
  • The difficulty level seems appropriate to the contest

Here are the approved APIs. Note that availability or capability might change

  1. Metropolitan Museum of Art Massive collection of artworks with search features
  2. Open Library Search by book, author, more. Retrieve covers and other data
  3. Currency API Get exchange rates for multiple currencies. Current and historical data
  4. Bored Find random activities to fight boredom
  5. QuickChart Build just about any kind of chart through a URL
  6. Free Dictionary Definitions, pronounciations, parts of speech, synonyms
  7. Open Food Facts Massive database of food from around the world with nutrition information
  8. Cheap Shark Steam and PC game search including current deals search
  9. Deck of Cards Online programmable card deck including hand management, card images
  10. Dungeons and Dragons Complete reference for spells, classes, and monsters
  11. Joke API Huge database of jokes with filter for inappropriate content
  12. Jeapordy Question Database Massive list of Jeapordy categories and questions
  13. Open Trivia Multiple choice trivia questions sortable by category, difficulty
  14. PokeAPI Get information about your favorite pokemon
  15. TV Maze Get information about TV shows
  16. I can has Dad Joke Random groan-worthy Dad jokes
  17. Newton Symbolic and arithmetic Math calculator
  18. Ball Don't Lie NBA data since 1979 including near real-time scores
  19. Meta Weather real-time and historical weather data
  20. Weather API A simpler but less detailed weather app

NOTE You might want to consider using the FireFox web browser when looking at the raw JSON, because it automatically formats JSON text, making it a lot easier to understand and examine.

Technology Resources

You should be able to do this project in any programming language you might already know. The easiest languages to do API work in are JavaScript and Python, but we will assist you with other languages. The language needs to support some sort of URL request (most do, but you might need to install a library.) You will also need to either have support for JSON objects, or be willing to parse the result JSON by hand. For example, Java does not have a built-in JSON library, but you can either add one, or use the String manipulation tools in the language to present an output.

You should be able to use replit or PythonAnywhere for hosting your program, so you will not need to install any programs on a school computer or your home machine.

Sample Code

To get you started, we'll look at the jsontest API. This is a simple API that returns a time stamp. Note that the time is relative to the server which seems to be somewhere in Europe, but that doesn't really matter. We're just looking for an easy API to practice with. Here is how to connect to that server in the most commonly used languages:

  • Python - APItest.py If you can use it, Python is one of the easiest languages for this project, as it already has strong support for reading a remote URL and parsing to JSON. If you use Python, the requests library is strongly recommended. It is already installed on pythonAnywhere, but you might need to install it in another setting. Note also that PythonAnywhere has a whitelist of sites you can go to. While most of the APIs are on that whitelist, it is possible you may not be able to get data from every source.
  • JavaScript APItest.html JavaScript / HTML also makes it quite easy to query an API and work with the resulting data. (after all, the JS in JSON stands for JavaScript.) It's particularly easy if you use the jQuery module as described in the sample project. The javaScript solution works anywhere, and is particularly easy if you want to work with images. You will also need to know a bit of HTML and CSS to make this work well.
  • Java APItest.java Java is a powerful language, but native support for this kind of work is a bit clunky. The example file has a function you can re-use to read a JSON file from any URL, and this function works with off-the-shelf Java. However, there is not a built-in JSON parser in Java. So for this example, I used an external library from Google called GSON. You will need to know how to add a library to your IDE or how to compile with an external Jar file (I have included a makefile for unix, and your coach should be able to help you modify that as needed.) For the GSON approach to work, you need to analyze the shape of the incoming JSON string and build a custom class to accept the data. It is tedious, but the resulting class can also have custom methods and attributes, making it quite powerful. (Welcome to Java.)
  • Other languages Most high school programs are using one of the three mentioned languages. If you are learning in another language (say C# or VB) contact your coach, and we'll see if we can put together a prototype you can work from. This project will be more difficult in C or C++, as they really don't have built-in networking, but we can help you if that's the language you want to work in. Note that in the scorekeeping, we will take into account the difficulty of the language you are using.

Scoring and Presentation

You will have at least a month to work on your project, with consultation from your coach and your high school teacher. You will submit your final project about a week before the CS day event, and we will score the projects before the event. You may be asked to prepare a presentation of your project, so you should assume that you'll need a two-minute quick overview of what your project does and how it works.

The scoring rubric will be released before the formal start of the competition, but will likely involve at least these components:

  • creativity - What clever thing did you do with the data set? What problem did you solve, or what fun tool did you make? How much of your team personality shines through the project?
  • documentation - How good is your planning and final documentation? Did you clearly delineate what you were trying to accomplish? Did you set up a reasonable schedule with clearly-marked milestones? Did you explain how to use the program?
  • coding style - Did you use a strong coding style? Did you comment well? Did you use sensible variable and function names? Did you use accepted programming practices?
  • complexity - Did you try to tackle a very simple problem with few parameters? Or did you add a substantial amount of features to make the program more flexible and usable? If you used a language without a lot of built-in features, how did you get it to work?
  • professionalism - Does the program work well without flaws and hiccups? Is it easy for users to learn and use?
  • aesthetics - You can use whatever type of user interface you want - command line with menus, standalone application, web site. But did you think carefully through these choices and use your chosen UI type to enhance the user experience?
  • presentation - Did you have a solid presentation that confidently shows the features of your program? Did you explain it well? Did you present well?
