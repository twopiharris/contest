Problem 10: T9 Predictive text

Background

It is possible (if tedious) to type on a standard telephone keypad. The t9 predictive text algorithm works by analysing the words that could be spelled using a particular sequence of numeric keys.

The first step is to determine all of the possible letter combinations that could be spelled with a particular numeric sequence. Then this list is checked against a dictionary. For this exercise, you only need to do the first part.

Given a string of numbers, you will determine ALL the possible letter combinations of those numbers. (Not simply the ones which spell words in English.)

For reference, here are the numbers on a telephone keypad and the corresponding letters:

0 no letters
1 no letters
2 A B C  
3 D E F  
4 G H I  
5 J K L  
6 M N O  
7 P Q R S
8 T U V  
9 W X Y Z

Input Expectations

Each line of input will be a string of numbers. There may be any number of values in that string, and there may be any number of lines. You may assume that the input contains only the values 2-9, as there are no letters on the zero or one key.

Please use "10in.txt" as the input file name.

Sample Input

523
74

Output Expectations

Output will consist of multiple lines per input. For each line of input, print ALL the possible character combinations (not just the ones that work as words.) Output should be one term per line, organized in alphabetical order.

Place a blank line between each group of outputs, to clarify a new set of responses has begun.
